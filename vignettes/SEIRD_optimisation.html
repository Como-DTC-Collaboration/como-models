Introduction

This document shows an application of the SEIRD model class within the como-models package. The SEIRD model can be used to gain insights from real COVID-19 data, such as extracting transmission parameters values. After fitting model to data, it can be used to make prediction of the pandemic. However, due to limited information, presence of errors and other problems, different sets of optimised parameters can fit the model to the data, achieving same objective value. This creates an issue of identifiability in parameters. Therefore, we have to be caution when intepreting optimisation results.

In this vignette, we want to show how optimisation is done with our package and the identifiabilty of parameters in the SEIRD model.

Optimisation on real London COVID-19 data

First, we would optimise the data on London COVID-19 data. The time period of data we used is before any intervention is implemented, lockdown in this case. Every intervention alters the parameters of the SEIRD model. Since we are fitting only one set of parameters, constant throughout the whole period, we chose to use London COVID-19 data before lockdown.

For the optimisation, we want to maximise its Poisson log-likelihood function, which is defined as below. It takes the average of both log-likelihood values of incidences and deaths. With \(L\) as the likelihood function, \(N\) as the size of the data, \(k\) as the actual data and \(\lambda\) as estimator of the SEIRD model, the log-likelihood function is

\[ \log L = \sum_{i=1}^N (k_i \log \lambda_i - \lambda_i) \]

RealData_LogLikelihoodFn <- function(parameters, model=SEIRD(), inc_numbers=0, death_numbers=0) {
  
  transmission_para <- list(beta=parameters[1],
                            kappa=parameters[2],
                            gamma=parameters[3],
                            mu=parameters[4])

  init_cond <- list(S0=parameters[5],
                    E0=parameters[6],
                    I0=parameters[7],
                    R0=parameters[8]) 
  transmission_parameters(model) <- transmission_para
  initial_conditions(model) <- init_cond
  
  times <- seq(0, length(inc_numbers), by = 1)
  out_df <- run(model, times)
  
  data_wide <- spread(out_df$changes, compartment, value)
  data_wide$Incidence <- abs(data_wide$Incidence) * population_size
  data_wide$Deaths <- abs(data_wide$Deaths) * population_size
  
  logIncidence <- log(data_wide$Incidence[-1])
  incidence_likelihood <- sum(inc_numbers * logIncidence - data_wide$Incidence[-1])
  logDeaths <- log(data_wide$Deaths[-1])
  death_likelihood <- sum(death_numbers * logDeaths - data_wide$Deaths[-1])

  (incidence_likelihood + death_likelihood)/2
}

Profile likelihood

To address the identifiability problem observed in optimisations of real data, an identifiability analysis is done using profile likelihood. The profile likelihood is the maximum likelihood value obtained by fixing a parameter of interest and optimising the remaining parameters. Here, the profile likelihood is constructed using synthetic data.

Simulate data

The data is simulated with values that provide sufficient time points before the pandemic peaks. This would replicate what is obtained from real data, such as the London data shown above.

Add noise to synthetic data

The synthetic data is then drawn from Poisson distribution with \(\lambda\) as the incidences and deaths of model simuated value, as shown in the equations below. \(C(t)\) and \(D(t)\) are number of incidences and deaths at time \(t\) respectively. while \(C^{*}(t)\) and \(D^{*}(t)\) are number of incidences and deaths with Poisson noise.

\[ C^{*}(t) \sim {\sf Pois}(C(t))\] \[ D^{*}(t) \sim {\sf Pois}(D(t))\]

Create error function

Poisson log-likelihood function is defined for optimisation of synthetic data. The profile likelihood is constructed for transmission parameters, while the initial conditions are fixed at values used to simulate the data.

                    fixed_parameter=FALSE, fixed_parameter_value=0) {
  
  transmission_para <- list(beta=parameters[1],
                            kappa=parameters[2],
                            gamma=parameters[3],
                            mu=parameters[4])
  if (fixed_parameter != FALSE){
    transmission_para[names(transmission_para) == fixed_parameter] <- fixed_parameter_value}

  init_cond <- list(S0=simulating_para[5],
                    E0=simulating_para[6],
                    I0=simulating_para[7],
                    R0=simulating_para[8]) # 1-sum(parameters[5:7])

  initial_conditions(model) <- init_cond

  logDeaths <- log(data_wide$Deaths[-1])
  death_likelihood <- sum(death_numbers * logDeaths - data_wide$Deaths[-1])

  (incidence_likelihood + death_likelihood)/2
}

Profile likelihood of \(\beta\) and \(\gamma\)

In some cases, identifiability issues arise when there are more parameters describing the model than the information can infer. In other words, parameters cannot be uniquely identified with limited data. When the number of parameters increase, parameters might become unidentifiable. Here, we first create a profile likelihood for two of the four transmission parameters in the SEIRD model.

Set ranges for error profile

constraint_ui <- rbind(diag(2))
constraint_ci <- c(rep(0,2))

profile_parameters <- c('beta', 'gamma')

beta_range <- c(seq(simulating_para[1], 0.4, by = -0.02), seq(simulating_para[1], 1.5, by = 0.02))
gamma_range <- c(seq(simulating_para[3], 0.1, by = -0.02), seq(simulating_para[3], 1.1, by = 0.02))

range_transmission <- data.frame(parameter=rep('beta', length(beta_range)), fixed_value=beta_range)
range_transmission <- rbind(range_transmission, data.frame(
  parameter=rep('gamma', length(gamma_range)), fixed_value=gamma_range))

Run optimisation

previous_param <- rep(0, 2)
profile_likelihood <- data.frame(parameter=character(), fixed_value=double(),
                                 likelihood_value=double(), optim_beta=double(),
                                 optim_gamma=double())
profile_likelihood$parameter <- as.character(profile_likelihood$parameter)

set.seed(0)
for (param_name in profile_parameters){
  err_value <- 0
  fixed_values <- range_transmission %>% filter(parameter == param_name)
  fixed_values <- fixed_values$fixed_value
  for (i in 1:length(fixed_values)){
    if (param_name == 'beta' & fixed_values[i] == simulating_para[1]){
      init_guess <- c(simulating_para[1], simulating_para[3])
    } else if (param_name == 'gamma' & fixed_values[i] == simulating_para[3]){
      init_guess <- c(simulating_para[1], simulating_para[3]) 
    } else {
      init_guess <- previous_param}

    result <- constrOptim(init_guess, LogLikelihoodFn2,
                          'NULL', constraint_ui, constraint_ci,
                          method = "Nelder-Mead",
                          control=list(fnscale=-1, reltol=1e-6),
                          model=model,
                          inc_numbers = testing_data_short$IncNoise,
                          death_numbers = testing_data_short$DeathNoise,
                          fixed_parameter = param_name, 
                          fixed_parameter_value = fixed_values[i])
    err_value <- append(err_value, result$value)
    previous_param <- result$par
    profile_likelihood[
      nrow(profile_likelihood) + 1,] <- c(
        param_name, fixed_values[i],
        result$value, result$par)
  }
}

While working through the optimisations for different fixed values of parameter of interest, the previously optimised parameters were used as initial guesses for the next round of optimisation. Due to the local optimiser we used, the range of parameters for the profile likelihood starts from the actual parameter value to the minimum and then from the actual parameter value to the maximum.
